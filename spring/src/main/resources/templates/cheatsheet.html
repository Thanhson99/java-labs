Core Syntax

The pieces you will meet first in almost every Java file.

Variable

A named value stored in memory.

int age = 25;
String name = "ThanhSon99";

Parameter

An input variable declared in a method signature.

public String greet(String name) {
    return "Hello, " + name;
}

Argument

The real value passed into a method call.

greet("Spring");

Method

A reusable block of logic attached to a class.

public int add(int left, int right) {
    return left + right;
}

Operators And Expressions

The symbols that combine values and build conditions.

Arithmetic

int total = price + tax;
int rest = total - discount;
int amount = count * size;

Comparison

score > 50
age <= 18
status.equals("READY")

Boolean Logic

isAdmin && isActive
isStudent || isAdmin
!isExpired

Ternary

String label = score >= 50 ? "pass" : "fail";

Method Anatomy

Read a method left to right in this order.

public ReturnType methodName(Type parameterName) {
    // method body
    return value;
}
  • public: who can call it
  • ReturnType: what it gives back
  • methodName: action name
  • Type parameterName: input definition
  • return: output value

Method Patterns

These patterns appear constantly in backend code.

Void Method

public void logAccess(String username) {
    System.out.println(username);
}

Overloaded Methods

public String greet() { ... }
public String greet(String name) { ... }

Static Utility Method

public static int max(int left, int right) {
    return Math.max(left, right);
}

Throws Clause

public String read(Path path) throws IOException {
    return Files.readString(path);
}

Control Flow

How the program chooses and repeats behavior.

if

if (score >= 50) {
    return "pass";
}

switch

return switch (region) {
    case "APAC" -> "Asia Pacific";
    default -> "Other";
};

for

for (int i = 0; i < items.size(); i++) {
    System.out.println(items.get(i));
}

for-each

for (String item : items) {
    System.out.println(item);
}

Collections And Streams

A lot of Java business code is just transforming collections safely.

List

List<String> names = List.of("A", "B");
names.get(0);

Map

Map<String, Integer> scores = Map.of("A", 90);
scores.get("A");

Stream Filter

users.stream()
    .filter(User::active)
    .toList();

Stream Map

users.stream()
    .map(User::email)
    .toList();

Classes, Records, Interfaces

Know when you want state + behavior versus a lightweight data carrier.

Class

public class BankAccount {
    private BigDecimal balance;
}

Record

public record TokenRequest(String username, String password) {
}

Interface

public interface NotificationClient {
    void send(String userId);
}

Enum

public enum DifficultyLevel {
    BEGINNER, INTERMEDIATE, ADVANCED
}

Exceptions And Validation

Backend code fails by design. Good code fails clearly.

Throw

if (amount.signum() < 0) {
    throw new IllegalArgumentException("amount must be positive");
}

Try / Catch

try {
    return objectMapper.readValue(json, User.class);
} catch (JsonProcessingException exception) {
    throw new IllegalArgumentException("invalid json", exception);
}

Checked Exception

Files.readString(path); // may throw IOException

Unchecked Exception

throw new IllegalStateException("user already exists");
