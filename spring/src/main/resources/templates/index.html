System Topology

Read the app as a chain of responsibilities, not as isolated files.

01

Client Edge

Browser, curl, or tests enter through public and protected HTTP endpoints.

02

Security Gate

JWT auth decides who can call user and system APIs.

03

Write Path

Service layer validates, writes to the main DB, and records audit state.

04

Event Fan-Out

Successful registration can publish to Kafka, RabbitMQ, or both.

05

Consumer Side

In-process listeners and the extracted notification service both consume the same event.

What This Backend Teaches

Security

JWT access tokens, refresh rotation, logout, logout-all, reuse detection.

Data

Primary user database, analytics database, Flyway migrations, transaction rollback.

Messaging

Registration events published to Kafka and RabbitMQ with shared consumer processing.

Testing

Controller tests, service tests, Testcontainers for PostgreSQL integration.
